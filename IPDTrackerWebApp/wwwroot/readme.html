Xamarin Hamburger Menu – Master Detail Pages

In Xamarin the fly out menu is constructed with a Master Detail Page project. This allows for a master root page that contains the menu in either in line or fly out (pop up) modes. So to begin create a new Xamarin cross-platform project and select the Master Detail project type.

This will create a project with a tabbed page and a number of view pages. We then want to add a new Master Detail Page called RootPage.xaml. To do this select the views folder and right click selecting Add > New Item. Then in the new item page select Xamarin.Forms and Master Detail Page

This will create 3 new pages (RootPage.xaml, RootPageMaster.xaml and RootPageDetail.xaml) and a model class (RootPageMenuItem.cs).  We want to reorganise the structure of these pages to better fit with our project.

To begin move the RootPageMenuItem.cs model class in to the Models folder and change it’s namespace to be in the Models namespace. You will notice that the constructor contains a TargetType setter that sets a default page to open when a new item is added to the menu collection, if one is not specified when it is instantiated. We don’t want or need that so comment it out.

Next we need to alter the start page in the App.xaml.cs constructor. It currently will load the MainPage from the project however it needs to be set to the new RootPage.

We then want to alter the RootPageDetail.xaml file so we can use it as our billing entries page. Begin by renaming the file to BillingPage.xaml and the class to BillingPage in both the xaml and xaml.cs files.

We will also need to create a BillingPageViewModel class in the ViewModel section and a BillingEntry data class in the Models section of our project.

You will notice that we have added the INotifyPropertyChanged interface so that when an item’s property is changed the UI is made aware of this and updates. As it stands the Items collection will not function this way. For the BillingPageViewModel make a copy of the ItemsViewModel and change it use the new BillingEntry data class.

We can now create a ViewModel from the RootPageMaster page. Start by adding the new RootPageMasterViewModel class to the ViewModels section of your shared project.

Then cut the view model class from the RootPageMaster.xaml.cs and paste it into the new ViewModel class we just created.

Once we have pasted in the view model code we need to modify it so that it has menu items for the Items Page, Billing Page and About Page. Ensure that you enter the appropriate typeof Page for each item.

This enables the navigation between the 3 pages using the hamburger menu. Next we want to allow for adding and editing of items and billing entries. There is already an ItemDetailPage and NewItemPage. We can use those and add in similar pages for billing entries. To allow us to use the ItemDetailPage for editing we will alter the pages labels into Entry boxes and add in two way binding so that we can modify the information back to the data store. We will also add the Update, Delete and Cancel buttons to the toolbar.

We also need the event handlers in the ItemDetailPage.xaml.cs.

These event handlers will use the MessagingCenter to send the update and delete requests back to the ViewModel. The Cancel event handler will simply pop the current edit page off the navigation stack returning to the previous page. Because the item is being edited directly we can’t use the actual item in the observable collection. Instead we will make a copy of it and edit that item. On updating we then transfer the data across to the list and allow the INotifyPropertyChanged eventhandler push an update to the UI.

Then in the ItemViewModel we need to add the registration for the MessagingCenter Update and Delete events.

IPDTrackerController.cs

using System;

using System.Collections.Generic;

using System.Linq;

using System.Threading.Tasks;

using IPDTrackerWebApp.Model;

using Microsoft.AspNetCore.Mvc;

using Microsoft.EntityFrameworkCore;

 

// For more information on enabling Web API for empty projects, visit https://go.microsoft.com/fwlink/?LinkID=397860

 

namespace IPDTrackerWebApp.Controllers

{

    [Route("api/BillingEntries")]

    public class IPDTrackerController : Controller

    {

        private readonly IPDContext _context;

 

        public IPDTrackerController(IPDContext context)

        {

            _context = context;

        }

 

        // GET: api/<controller>

        [HttpGet]

        public IEnumerable<BillingEntry> Get()

        {

            return _context.BillingEntries;

        }

 

        // GET api/<controller>/5

        [HttpGet("{id}")]

        public async Task<IActionResult> Get(string id)

        {

            Guid _Id = new Guid(id);

 

            if (!ModelState.IsValid)

            {

                return BadRequest(ModelState);

            }

            var billingEntry = await _context.BillingEntries.SingleOrDefaultAsync(e => e.Id == _Id);

 

            if (billingEntry == null)

            {

                return NotFound();

            }

 

            return Ok(billingEntry);

        }

 

        // POST api/<controller>

        [HttpPost]

        public async Task<IActionResult> Post([FromBody]BillingEntry entry)

        {

            if (!ModelState.IsValid)

            {

                return BadRequest(ModelState);

            }

 

            _context.BillingEntries.Add(entry);

            await _context.SaveChangesAsync();

 

            return CreatedAtAction("Get", new { id = entry.Id }, entry);

        }

 

        // PUT api/<controller>/5

        [HttpPut("{id}")]

        public async Task<IActionResult> Put([FromBody]BillingEntry entry)

        {

            //Guid _Id = new Guid(id);

 

            if (!ModelState.IsValid)

            {

                return BadRequest(ModelState);

            }

 

            //if (_Id != entry.Id)

            //{

            //    return BadRequest();

            //}

 

            _context.Entry(entry).State = EntityState.Modified;

 

            try

            {

                await _context.SaveChangesAsync();

            }

            catch (DbUpdateConcurrencyException)

            {

                if (!BillingEntryExists(entry.Id))

                {

                    return NotFound();

                }

                else

                {

                    throw;

                }

            }

 

            return NoContent();

        }

 

        // DELETE api/<controller>/5

        [HttpDelete("{id}")]

        public async Task<IActionResult> Delete(string id)

        {

            Guid _Id = new Guid(id);

 

            if (!ModelState.IsValid)

            {

                return BadRequest(ModelState);

            }

 

            var billingEntry = await _context.BillingEntries.SingleOrDefaultAsync(e => e.Id == _Id);

            if (billingEntry == null)

            {

                return NotFound();

            }

 

            _context.BillingEntries.Remove(billingEntry);

            await _context.SaveChangesAsync();

 

            return Ok(billingEntry);

        }

 

        private bool BillingEntryExists(Guid id)

        {

            return _context.BillingEntries.Any(e => e.Id == id);

        }

    }

}

From a MasterDetailPage sample Xamarin project with an Azure backend copy the AzureDataStore class. This will enable us to connect to our azure data service. As the old IDataStore interface uses boolean returns so the azure file needs to be modified to return integers. We will return the web status code or an error code.

AzureDataStore.cs

using System;

using System.Collections.Generic;

using System.Net.Http;

using System.Text;

using System.Threading.Tasks;

using Newtonsoft.Json;

using Plugin.Connectivity;

using IPDTracker.Models;

 

[assembly: Xamarin.Forms.Dependency(typeof(IPDTracker.Services.AzureDataStore))]

namespace IPDTracker.Services

{

       public class AzureDataStore : IDataStore<BillingEntry>

       {

              HttpClient client;

              IEnumerable<BillingEntry> items;

 

              public AzureDataStore()

              {

                     client = new HttpClient();

                     client.BaseAddress = new Uri($"{App.AzureBackendUrl}/");

 

                     items = new List<BillingEntry>();

              }

 

              public async Task<IEnumerable<BillingEntry>> GetItemsAsync(bool forceRefresh = false)

              {

                     if (forceRefresh && CrossConnectivity.Current.IsConnected)

                     {

                           var json = await client.GetStringAsync($"api/billingentries");

                           items = await Task.Run(() => JsonConvert.DeserializeObject<IEnumerable<BillingEntry>>(json));

                     }

 

                     return items;

              }

 

              public async Task<BillingEntry> GetItemAsync(string id)

              {

                     if (id != null && CrossConnectivity.Current.IsConnected)

                     {

                           var json = await client.GetStringAsync($"api/billingentries/{id}");

                           return await Task.Run(() => JsonConvert.DeserializeObject<BillingEntry>(json));

                     }

 

                     return null;

              }

 

              public async Task<int> AddItemAsync(BillingEntry item)

              {

                     if (item == null || !CrossConnectivity.Current.IsConnected)

                           return 404;

            item.DateModified = DateTime.Now;

                     var serializedItem = JsonConvert.SerializeObject(item);

 

                     var response = await client.PostAsync($"api/billingentries", new StringContent(serializedItem, Encoding.UTF8, "application/json"));

 

                     return (int)response.StatusCode;

              }

 

              public async Task<int> UpdateItemAsync(BillingEntry item)

              {

                     if (item == null || item.Id == null || !CrossConnectivity.Current.IsConnected)

                           return 404;

            item.DateModified = DateTime.Now;

            var serializedItem = JsonConvert.SerializeObject(item);

                     var buffer = Encoding.UTF8.GetBytes(serializedItem);

                     var byteContent = new ByteArrayContent(buffer);

 

                     var response = await client.PutAsync(new Uri($"api/billingentries/{item.Id}"), byteContent);

 

                     return (int)response.StatusCode;

              }

 

              public async Task<int> DeleteItemAsync(string id)

              {

                     if (string.IsNullOrEmpty(id) && !CrossConnectivity.Current.IsConnected)

                           return 404;

 

                     var response = await client.DeleteAsync($"api/billingentries/{id}");

 

            return (int)response.StatusCode;

 

        }

       }

}

The Azure Client and SQLite Azure client link normally takes care of that. At this point, due to project incompatibilities, it isn’t possible. Instead we can add a simple sync module that will upload any stored items that we have cached.

SyncService.cs

using IPDTracker.Models;

using System;

using System.Collections.Generic;

using System.Diagnostics;

using System.Linq;

using System.Text;

using System.Threading.Tasks;

using Xamarin.Forms;

 

namespace IPDTracker.Services

{

    public static class SyncService

    {

        public static IDataStore<BillingEntry> LocalDataStore =>

            DependencyService.Get<IDataStore<BillingEntry>>() ?? new LocalDataStore();

        public static AzureDataStore AzureDataStore = DependencyService.Get<AzureDataStore>();

 

        public static async Task SyncAsync()

        {

            var items = await LocalDataStore.GetItemsAsync(true);

            if(items.Count<BillingEntry>() != 0)

            {

                foreach(var item in items)

                {

                    item.DateModified = DateTime.Now;

                    try

                    {

                        await AzureDataStore.AddItemAsync(item);

                        await LocalDataStore.DeleteItemAsync(item.Id.ToString());

                    }

                    catch

                    {

                        Debug.WriteLine(item.Id.ToString() + " failed to sync.");

                    }

                }

            }

 

            await Task.CompletedTask;

        }

    }

}
